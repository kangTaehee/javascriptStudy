자바스크립트

  1. 단일 스레드 실행구조
  2. 어느 한순간에는 단 하나의 작업만 실행할 수 있다.

호이스팅 : 메모리에 공간을 미리 할당한다.

  1. 전역 실행 컨텍스트 만들어진다
  2. 선언적 방식의 함수가 먼저 실행 컨텍스트의 메서드 형태로 만들어진다
  3. var 키워드가 사용된 변수가 미리 만들어진다. = 값이 할당 되진 않는다

실행 컨텍스트 Execution Context

  1. 자바스크립트 코드가 실행되는 환경 = 일종의 객체
  2. 처음 자바스크립트 코드가 실행될 때 전역 실행 컨텍스트에서 실행된다
  3. 함수가 호출될 때마다 새로운 실행 컨텍스트가 생선되어 스택에 추가되고, 제어권은 새롭게 생성된 컨첵스트로 이동한다.

var

var를 여러번 사용하여 변수를 할당하는것은 = 값을 새로 부여한다.

산술 연산자의 암시적 형변환

1+'2';//12 더할때는 문자열이 있으면 둘다 문자열로 인식한다. typeof = string

1-'2';//-1 뺄때는 문자열이 있어도 number로 인식한다. typeof = number

parseInt('123abc'); // 123

parseInt('abc123'); // NaN

parseInt()

문자열의 앞부분이 숫자로 시작할 경우 변환할 수 있는 부분까지 변환한다.

기본 리턴값 : NaN

parseInt(); // NaN

parseInt('abc'); // NaN

Number()

전체 문자열을 숫자로 변환할 수 있는 가를 판단한다.

기본 리턴값 : 0

Number(); // 0

Number('abc'); // NaN

NaN값에 대한 == 비교연산자 결과

객체의 값을 비교하는 경우 좌우 각 객체의 참조하는 메모리 주소가 같은지를 비교한다.

Number.NaN; //NaN

var a=parseInt('abc'), b=parseInt('abc');

a==b;//false

Number.NaN == Number.NaN; //false 각 객체의 주소값이 다르다. (여기서 false는 NaN이 아니라는 의미 이다)

결과 이유 : == 비교 연산이 기본 타입인 number값이 Number 객체로 암시적 형변환된 후에 수행되기 때문이다.
